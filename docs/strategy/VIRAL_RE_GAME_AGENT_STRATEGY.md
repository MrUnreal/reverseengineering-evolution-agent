# Viral RE Game Agent Strategy

## Positioning

**Category:** AI-assisted reverse engineering copilot for game systems mapping.

**One-liner:**
Turn opaque game binaries into confidence-scored architecture maps and shareable tactical insights in minutes.

## ICP (Ideal User Profiles)

1. Security researchers validating anti-cheat behavior.
2. RE learners who need guided workflows and evidence trails.
3. Modding/tooling communities needing safe compatibility analysis.
4. Technical creators who want visual artifacts for educational content.

## Product Pillars

1. **Speed-to-insight**
   - First useful map in < 10 minutes.
2. **Evidence-first output**
   - Every claim tied to signals and confidence.
3. **Visual clarity**
   - Dashboard-native outputs, not giant text dumps.
4. **Reproducibility**
   - Same input -> same analysis trace.

## Viral Mechanics

### 1) Shareable Insight Cards
- Auto-generate cards for:
  - subsystem map
  - anti-cheat vectors
  - protocol opcode coverage
  - confidence delta over iterations
- Embed badge: `Generated by RE Agent`.

### 2) Public Challenge Packs
- Weekly mini-binaries/challenges.
- Users submit analysis JSON.
- Agent scores coverage/accuracy.
- Public leaderboard.

### 3) Community Rule Packs
- Signature packs (opcode, anti-cheat, UI/rendering clues).
- Contributors get attribution + impact metrics.
- Most adopted packs become "recommended defaults".

### 4) Case-study Growth Loop
- Publish short, reproducible postmortems:
  - baseline manual time
  - agent-assisted time
  - discoveries unlocked
- Template each post for easy repost on X/Reddit/YouTube.

## MVP to Viral (90-Day Path)

### Phase A (Days 1-30): Trust Engine
- Normalize findings schema.
- Add deterministic run metadata.
- Add quality gates for confidence claims.
- Ship `demo mode` with no binary needed.

### Phase B (Days 31-60): Distribution Engine
- Publish hosted dashboard snapshots.
- Add export bundle (`.zip` report + card images + markdown summary).
- Launch challenge #1 with leaderboard.

### Phase C (Days 61-90): Community Engine
- Open rule-pack registry.
- Add contributor analytics.
- Run recurring "RE Agent Friday" releases and recap threads.

## Metrics That Matter

### Product
- Time-to-first-insight
- Coverage score (subsystems/opcodes/handlers)
- Confidence calibration error

### Growth
- Share rate per completed analysis
- Invite-to-activation conversion
- Weekly active contributors (rule packs)
- Challenge participation rate

### Retention
- % users running second analysis in 7 days
- # saved templates reused per user
- Cohort retention by ICP

## Differentiation

1. Confidence-scored, evidence-linked claims (not black-box summaries).
2. RE workflow automation + visualization in one loop.
3. Pattern memory that compounds with community submissions.

## Execution Backlog (Top 10)

1. Unified CLI wrapper for all analyzers.
2. Canonical JSON schema + validator.
3. Dashboard "evidence drill-down" panel.
4. Export share cards (PNG + markdown).
5. Demo dataset and one-click run.
6. Rule-pack loader and versioning.
7. Challenge mode + scoring service.
8. GitHub Actions smoke pipeline.
9. Telemetry for anonymous product metrics.
10. Public docs site with 3 flagship case studies.

## Guardrails

- Keep analysis educational/defensive and policy-compliant.
- Avoid publishing sensitive exploit details or harmful automation.
- Require explicit confidence labels on all high-impact claims.
